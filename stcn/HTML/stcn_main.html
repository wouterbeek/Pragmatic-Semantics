STCN, Short Title Catalogue Netherlands

The STCN contains 139.817 entries. These have been parsed for the following KMC codes. This page documents (1) what data has been parsed and turned into RDF(S), (2) what other datasets the parsed STCN data has been linked to, and (3) some of the undocumented values we came across during the process.

KMC 1100 - Year

139.817 PPNs (100%) have a year value. Of these 126.598 (91%) are exactly dated and 12.219 (9%) are approximately dated. The year values that occur in the STCN range from 1473 (earliest) to 1847 (latest).

KMC 1200 - Typographic property

There are 419.711 typographic property values in the STCN. All of them are mapped to appropriate resources. For this conversion we have made decisions on how to represent undocumented values (see the list below).

  1. Some PPNs use this code to encode some completely different property. E.g. PPN 14162843X, PPN 14162843X, PPN 314559655.
  2. '1' (the number between 0 and 2) seems to occur to be a typo of 'l' (the letter between 'k' and 'm'). E.g. PPN 321339320.
  3. Uppercase variants of documented lowercase versions occur. E.g. PPN 298324997, PPN 305305972.
  4. Some PPNs have the undocumented value '5'. E.g. PPN 271591978.
  5. Some PPNs have the undocumented value '9'. E.g. PPN 301032831.
  6. The value 'w.' (with the dot) seems to occur as a variant of 'w' (without the dot). E.g. PPN 204513316.
  7. 'v' occurs in the STCN documentation, but not in the GGC documentation.
  8. 'b)' seems to be a variant of 'b'. E.g. PPN 261654624.

KMC 1500 - Language

The STCN and GGC do not distinguish languages from language families.

139.801 PPNs have a language value. All of these have at least a value for the language of the work itself. 23.586 of these PPNs (with known language) also have a value for a language from which they were translated. 827 of these PPNs (with known language) also have a value for the language via which they were translated. 53 languages occur in the STCN.

Language codes have been linked to languages in the ISO standards 639-1, 639-2, 639-3; and to language families in the ISO standard 639-5. The URIs are drawn from the Lexvo dataset.

The Lexvo dataset links languages and language families to several other resources. For example the Dutch language is linked to the following resources:

  • owl:sameAs http://dbpedia.org/resource/Dutch_language
  • owl:sameAs http://psi.oasis-open.org/iso/639/#nld
  • owl:sameAs http://www.lingvoj.org/lang/nl
  • owl:sameAs http://www.mpii.de/yago/resource/Dutch_language
  1. The following translations have been made to bring the STCN/GGC language codes in agreement with ISO standards (STCN lhs; ISO rhs):
    1. arm --- hye
    2. chi --- cmn
    3. dmh --- gmh
    4. fri --- fry, ISO 639-3 distinguishes between the West-Frisian (spoken in The Netherlands) and the North-Frisian language (spoken in Germany). West-Frisian has been chosen as the translation in all cases.
    5. grk --- grc
    6. hon --- hun
    7. jap --- jpn
    8. ned --- nld
    9. per --- fas
    10. zwe --- swe
  2. Some PPNs have language families encoded i.o. individual languages:
    1. Prakrit (pra) used in PPN 303735422
    2. Iroquoian (iro) used in PPN 305228218
    3. Creole or Pidgin (crp) used in PPNs 301776601, 23838229X
  3. There are 84 language codes that are documented by GGC but that are not in the ISO standards. These language codes do not occur in the STCN.
  4. There are multiple ways of specifying that the (exact) language is unknown.

KMC 1700 - Countries

139.778 PPNs (99.97%) have a country value. 7.574 PPNs (5.4%) have, in addition to that, a value for the actual country in which they were published (which is different from the country that is mentioned in the publication itself). 34 countries occur in the STCN.

Country codes have been completely (all PPNs) linked to countries in the ISO standard 3266-1. The URIs are drawn from the Lexvo dataset.

The Lexvo dataset links countries to several other resources.

  1. Czechoslovakia no longer exists (as of 1992), so country code cs is currently obsolete.
  2. There are 21 country codes that are documented by GGC but that are not in the ISO standards. These country codes do not occur in the STCN.

KMC 3000 and KMCs 3011 through 3019

Primary and secondary authors are linked to their Picarta entries. Picata does not have a linked dataset yet, so we used page scraping for this. Picarta is still a good linker, since it uses PPN codes for author which are unique identifiers.

PPN statistics related to authors:

  1. 93.724 PPNs (67%) have at least one author.
  2. 86.324 PPNs (62%) have a primary author.<\li>
  3. 70.040 PPNs (50%) only have a primary author.<\li>
  4. 23.684 PPNs (17%) have a secondary author.<\li>
  5. 7.400 PPNs (5%) only have a secondary author.
  6. 2.244 PPNs (1.6%) are written under pseudonym.

Author statistics:

  1. The STCN contains 22.814 authors.
  2. All authors were found in Picarta (which is evident, since Picarta contains the STCN).
  3. 578 authors (2.5%) were linked to DBpedia entries.
  4. 445 authors (2.0%) are pseudonyms of authors within the STCN dataset.

Using the redactiebladen parsed created by data2semantics / VU

Installation

  1. Retrieve the codebase: git clone https://github.com/Data2Semantics/STCN.git.
  2. Install the latest SWI-Prolog development release (tested on 6.3.10).
  3. Create a directory .STCN in your home folder and copy the redactiebladen text file (uncompressed) there. Give the file the name redactiebladen.txt.
  4. In the console run the command swipl -s stcn.pl.
  5. In the SWI-Prolog console run the following: stcn_web:stcn_redactiebladen_web(_)..
  6. Open a Web browser on http://localhost:5000/prasem/. From the Web interface you can run various commands (described in the next section) to check the parsing process.

Detailed documentation

Detailed documentation was written for each of the files in the codebase. For the files covering the respective KMCs, the documentation includes all the exceptions we came across while building the parser, as well as complete tables of legal and illegal values.

The detailed documentation is accessed by running the documentation command from the Web-interface.

Displaying dynamic tables in the detailed documentation

If you want to make the dynamic tables visible in the plDoc documentation server, you need to add the following clause to the take_block predicate in module pldoc/doc_wiki.pl of your SWI-Prolog installation. (This code cannot be included in SWI-Prolog releases because of the security implications of running arbitrary code on the documentation server.

take_block(
  [_-['!', '!', w(Module), '!', '!', w(Predicate), '!', '!', w(Arity0), '!', '!'] | Lines],
  _BaseIndent,
  table([class(wiki)], Rows),
  Lines
):-
  atom_number(Arity0, Arity),
  findall(
    tr(Cells),
    (
      length(Vars, Arity),
      Call =.. [Predicate | Vars],
      call(Module:Call),
      findall(
        td([w(Atom)]),
        (
          member(Var, Vars),
          (atomic(Var) -> Atom = Var ; term_to_atom(Var, Atom))
        ),
        Cells
      )
    ),
    Rows
  ).

Commands run from the Web interface

The Web interface can be run locally (as described above) or a sample version of it can be run only at http://http://semanticweb.cs.vu.nl/prasem/.

The Web interface shows a console in which commands can be given. The following lists some of the commands that can be used w.r.t. the STCN dataset.

help
Gives an list of the avialable commands.
stcn_statistics
Displays the statistics for the STCN parser. This command can be executed during and after the parsing process in order to show partial or complete results.
schema(stcn)
Displays the schema of the STCN dataset, i.e., the concepts that structure it.
ppn(NUMBER)
Shows a visual representation of the information the dataset contains regarding a specific PPN NUMBER.
sparql
Opens an input field for entering a SPARQL query.
